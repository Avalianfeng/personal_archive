## 1. 一句话立意

> **我本质不是在建立一个人的档案,而是在讲我自己的分析体系、讲自己的世界观——用我对人的思考和认识建立分析模型。**(你的原话,手填卷末批注)

「人物档案」是这套分析体系的**载体和验证场**,不是目的本身。读者最终通过档案认识的不只是「被记录的那个人」,也是「这套看人的方法」。

## 3. 立意的三个推论

### 3.1 目录即世界观

[02-档案目录.md](02-档案目录.md) 的 §0–§13 不是「问卷分类」,而是「**看懂一个人需要看哪些维度**」的本体论。它回答的是认识论问题:Meta 语境层(§0)承认「任何自述都发生在具体语境里」;正文十三章是观察一个人的完整透镜;§13 综合观察承认「人是拼出来的整体,不是条目之和」。

### 3.2 金样优先于工程

体系是否成立,唯一的检验是:**它能不能把一份真实语料(你自己的手填卷)转写成一份「读完像认识了一个人、同时看得见分析方法」的报告**。所以分析报告金样([samples/report-v1.md](samples/report-v1.md))先于一切代码——prompts、题库、引擎、Web 全部从定稿金样反推,而不是反过来。

### 3.3 分析者在场

报告采用**分析者视角**(已定):分析与档案相辅相成,不刻意用「客观档案体」约束。〔推断〕这意味着这套体系的输出天然带署名性——它是「某个分析体系对一个人的解读」,而非伪装中立的机器记录。

## 4. 实现路径(金样驱动,唯一路线)

```text
① 输入金样      samples/intake-v1.md        ✅ 已完成并冻结
② 输出金样      samples/report-v1.md        ← 当前位置:AI 初稿 → 你二次定稿
③ 反推 prompts  extract / synthesize 提示词   以定稿金样为裁判
④ 引擎闭环      答案 → PersonModel → 渲染     代码迁回,跑通至少一章
⑤ Web 采集      题库分册 → 表单/沉浸式皮肤     明确为后期
```

每一步的完成标志只有一个:**上一步的产出能裁判下一步的质量**。不允许跳步(教训:之前「不断优化问题、却不知道最终生成什么」,见 gpt 评审)。

## 5. 不变原则

1. **档案是关于人的,不是关于问题的。** 每一节回答「了解这个人」,而非「第几题放了什么」。
2. **建档优先于测评。** 正文不以「某某型人格」为主体;类型学标签仅可进附录作参考。
3. **详尽档案本体唯一。** 专项问卷(技能、收入等)外挂到相关三级条;简略模式从详尽中裁剪,不另建短目录。
4. **存储可分客观/人格,阅读应像一个人。** 不按「客观卷/人格卷」分章。
5. **手填原版冻结。** [samples/intake-v1.md](samples/intake-v1.md) 的答案与 `{}` / `<<>>` 批注不再修改。
6. **设计顺序不可颠倒。** 即 §4 的五步路线。
7. **贫语料假设(2026-06-11 增)。** 金样填写者的语料丰富度是金样的**特权,不是体系的前提**。普通用户不会坐下来打一小时字讲故事——体系的默认输入是「选项 + 简短回答」,深度必须由两样东西萃取:**追问设计**(动态、低成本的二轮短问)和**提示词**(从每一个简单答案背后读出蕴藏的内容)。「问题+选项得倾向底色」的传统问卷是我们的下限兼容,不是我们的形态;但「依赖长篇自述」也不是我们的形态。路线 ③ 反推 prompts 时,必须同时回答:**同一条三级条,在贫语料输入下如何写出不缩水的档案语言?**
8. **防语料污染(2026-06-11 增)。** 本体系第一版完全生长在单一 Persona(我自己)的语料和表达风格之上,存在把「我的思维特征」误固化为「体系本身」的风险——对分析者(AI)同样如此,大量同源语料会塑形其解读习惯。对策:体系成型后**强行引入外部校准**——别人的思考与认识方法、经典哲学与心理学的思考模型、以及第二个真实 Persona 的全流程金样。在此之前,所有从我的语料归纳出的「分析模式」都只享有候选地位。

## 6. 「足够」的验收标准(精简版)

报告写到什么程度算成立——四层全过即可,不追求完备:

1. **陌生人五问**:读完后陌生人能不靠猜回答——他是谁/处在什么阶段、最近在意什么、看重什么不能碰什么、如何与人相处、往哪走怕什么期待什么。
2. **跨章一致**:至少 2–3 条主题线在多章互相印证(如「对思考的依赖与不坚定」应贯穿处境、自我认知、决策)。
3. **材料率**:核心章(§2、§4、§5、§7、§12)≥80% 三级条有材料,全文 ≥60%;空缺写「尚未记录」,不阻断。
4. **快照诚实**:标明快照日期、时间尺度(此刻/一贯/混合)、填写语境。

(完整版本见 [archive/project-guide.md](archive/project-guide.md) §2,该文档已归档,验收标准以本节为准。)

## 7. 体系与自我分析的关系(2026-06-11 定)

**搭建分析体系 ≠ 分析我自己**,但二者的次序已定:**先把对我自己的分析完成,再考虑其他**。自我分析是体系的第一个完整用例,不是体系的全部;它的产出(report-v1 定稿)既是档案,也是体系第一次运转的实验记录。

同时承认一个已观察到的现象:填写者阅读自己的分析报告时,会被拉入自我认识而难以保持对「分析行为本身」的结构性思考。这不是缺陷,是这类工作的固有双重性——处理方式是**显式分轨**:读报告时做被分析者(改内容),读立意与设计文档时做体系构建者(改结构),不在同一次工作中混用两个身份。
