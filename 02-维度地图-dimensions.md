## 使用说明

本文档分四块，阅读时注意标记：

| 标记 | 含义 |
| --- | --- |
| **【Meta】** | 档案语境层，影响解读与权重 |
| **【维度】** | 认识论三级条（引擎/题库坐标） |
| **【设计说明】** | 给产品/实现的内部备注 |
| **【附录】** | 有材料才展开 |

层级约定：**一级 = 大章 · 二级 = 节 · 三级 = 条**（三级是「应了解的具体方面」，不是问卷题目）。

**与文件群的关系**：§0–§13 是维度本体；§14 映射到 `samples/persona-<id>/` 各侧面文件。文件群是**阅读分区叠加层**，不替换三级条。

## 核心原则

{标记为BACK2,阅读完我的批注后追加或者修改原则}

1. **档案是关于人的，不是关于问题的。**
2. **建档优先于测评。** 类型学标签仅可进附录。
3. **所有采集插件内容必须能归入某一三级条。**
4. **存储按维度分桶，阅读按文件群分区。** 见 §14。

### 0.2 Frame · 你如何认识这次问答

{提问不一定要只有一次，可以完成问卷后再问一下，然后也可以只在最后问，不仅仅是这个问题本身，如果前后不一致也有考虑的空间。我此刻的想法是如果前后不一致后一次的可信度更好}

{可以结合这个：你是从哪里了解到本次测试，这会有很大的参考价值，这些思路来自注册一些网页后会有的一些提问，可以说是有很大相似度的}

| 三级条 | 说明 |
| --- | --- |
| 0.2.1 建立动机 | 认真建档 / 随便看看 / 帮他人了解我 / 娱乐玩玩 / 其他 |
| 0.2.2 与「测试」的关系{问题问法改一下，比如在问的时候明确说：进入人格测试阶段，然后问，或者“你对其中的人格测试内容的看法”这样子} | 主要来测人格 / 想全面了解自己 / 只是填着玩 / 不确定 |
| 0.2.3 诚实度预期 | 尽量真实 / 部分美化或表演 / 当角色扮演 |
| 0.2.4 时间尺度{这一点过分主观，可以删掉} | 此刻的我 / 一贯的我 / 混合 |
