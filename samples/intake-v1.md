# 人物档案建档问卷 V1

## 产品定位

本卷覆盖 **「人」的详尽档案**（对应 [理想人物档案目录 V1](../archive-catalog-v1.md) §0–§12 及附录 §A）。

| 范围 | 说明 |
| --- | --- |
| **本卷（人）** | 建档、自我认知、关系、情感、价值观、轨迹、未来等；个人档案与人格叙述 **以本目录完整内容为基准** |
| **专项插件（外挂）** | 技能细目、收入、行业问卷等 **单独采集**，挂载 §9 等相关条，不并入本卷题量 |
| **人格深潜等** | 沉浸式探测独立进行，摘要进附录或汇入正文，不以插件分册替代目录 |
| **简略模式** | 从同一份详尽档案 **选取** 呈现，不另建更短目录 |

## 本文档性质（重要）

| 属性 | 说明 |
| --- | --- |
| **当前状态** | **首份完整手填原版**（Persona 样例），含填写者答案与 `{}` / `<<>>` 批注 |
| **不是** | 上线用标准问卷、题库 JSON、或可直接导入 Web 的最终题面 |
| **后续产出** | 分章优化后，另立：`questions/` 分册题库、`prompts/` 分析提示词、理想报告样章等 |

> **用途**：验证目录是否承载真实叙述；为 Processor / Renderer 提供黄金素材。  
> **填写说明**：在选项前打勾，或在「答」后的空行书写；可跳过任何题目；第 13 章「综合观察」由系统生成，无需填写。
> “{}”是我进行自我填写的时候的备注内容，可能有Q标注存在问题，后面细细思考，或者直接问，需要你解释，具体以“{}”内容为主,F表示功能设计，可能是文本工程，可能是web功能
> “<<>>”是我自己的考虑和想法，不一定关系到项目本身
