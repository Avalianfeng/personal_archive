# 产物 A/B 提示词与结构改进建议
> 版本：v1 · 日期：2026-06-14
> 范围：product-a（分析报告）提示词、product-b（人物档案）提示词 + 目录结构
> 不含：评估问卷、隐私双版本、插件体系扩展

---

## 一、产物 A · 分析报告

### 1.1 现有问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 「第三人称或第二人称均可」——口吻未定 | 模型随机选择，用户读到第三人称会有距离感，减弱自我认识的冲击力 |
| 2 | 敏感内容（向死倾向、未解家人议题、深度痛点）与一般内容混排 | 用户没有心理预期直接读到，体验粗暴；也阻碍将来的对外分享场景 |
| 3 | 写作规则未规定哪些内容「必须进正文」vs「必须单独处理」 | 模型自由裁量，金样以外的用户可能得到过重或过轻的处理 |

---

### 1.2 改动建议

#### 改动 1：口吻规则——强制第二人称

**现有写法（System 写作规则第 1 条）：**
```
第三人称或第二人称均可，口吻克制、观察式，有少量分析者存在感
```

**修改为：**
```
统一使用第二人称（「你」），不使用姓名或「他/她」。
口吻：克制、观察式，带少量分析者存在感——像一个认识你很久的人
在说一件你自己也知道但没有说出口的事。
```

**理由：** 产物 A 的唯一读者场景是用户看自己的报告。第二人称让「你是一个把思考当作居所的人」这类句子直接落地，第三人称则会产生「在读别人」的疏离感，削减自我认识的效果。


#### 改动 2：增加内容分级规则

在写作规则末尾新增第 7 条：

```
7. 【内容分级】
   正文（第 1–7 章）只写「气质 / 驱动 / 思维 / 关系模式 / 成长课题 / 发展方向」。

   以下类型内容，单独放入第 8 章「总结」之后的独立节：
   【深度观察】
   触发条件（满足任意一项即进入该节）：
   - 填写者自述存在朝向死亡或毁灭的内在倾向
   - 长期深度痛点尚未命名或处理
   - 家人 / 原生家庭议题明确「情感层未结算」且填写者自述未想清楚
   - 其他填写者在 L1 中被标注「最需要后续对话」的内容

   【深度观察】节的写法：
   - 不做临床判断，不贴标签
   - 以「我们注意到……」开头，观察而非定义
   - 结尾注明：「这部分内容比正文更接近你的私域，
     读的时候可以慢一点。」
```

**理由：** 深度内容放在独立节有两个好处：一是用户读正文时不会毫无准备地遇到最重的东西；二是将来如果做「分享版」报告，可以直接裁掉这一节，不需要重新生成。

```

2. 允许归纳与解释，但不捏造 L1 中没有依据的内容。

   以下类型内容，单独放入第 8 章之后的独立节【深度观察】：

   【深度观察】节的写法：
   - 不做临床判断，不贴标签
   - 以「我们注意到……」开头，观察而非定义
   - 结尾注明：「这部分内容比正文更接近你的私域，读的时候可以慢一点。」
```

## 二、产物 B · 人物档案

### 2.1 现有问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 自述内容和可观察事实未区分存入档案 | 档案变成用户自我叙事的整理版，失去旁观者客观性 |
| 2 | 稳定特征和动态特征混在同一结构里 | 每次更新要面对全文；阅读时重心分散 |
| 3 | 目录中「娱乐与内容偏好」和「金钱花费方向」各自重复出现 | 生成时容易重复填充，信息冗余 |

---

### 2.2 改动建议

#### 改动 1：加入自述 / 观察区分原则

在 System 写作规则新增第 7 条：

```
7. 【来源标注原则】
   档案每一条信息必须隐性区分来源，写法如下：

   - 可观察事实（行为、事件、可验证信息）：直接陈述
     示例：「社交圈小，不主动经营关系。」

   - 填写者自述（对自己的解读、感受、判断）：加来源标注
     示例：「自述将孤独视为『更高价值的自我提升空间』。」
             「自述感到痛苦，但此刻已能较自然地谈及。」

   - 分析者归纳（从多处材料推导，非单一自述）：加「观察：」前缀
     示例：「观察：家庭情感地图重心在祖辈而非父母辈。」

   禁止：把填写者的自我解读直接写成客观陈述。
   错误示例：「孤独对他而言是正资产。」（这是解读，不是事实）
   正确示例：「自述孤独是『人生应该有的更高价值的自我提升空间』。」
```

**为什么这是原则问题：** 档案的长期价值在于它是可以被下一个分析者拿去重新解读的原材料。如果自述和事实混写，下一轮分析就会在别人的解读上继续加解读，信息质量会层层衰减。

---

#### 改动 2：目录结构重构——稳定层 / 动态层分离

**现有结构问题：** §0–§8 是单一平铺，稳定信息（价值观、思维方式、关系模式）和动态信息（当前张力、近期事件、正在探索的问题）混在同一层级，更新成本高，阅读重心模糊。

**新结构：** 拆成两个独立文件。

---

##### 文件一：核心档案（稳定层）

> 更新频率：数月到数年一次，仅在长期特征有实质变化时修改
> 目的：「这个人是谁」的骨架

```
# 核心档案（稳定层）

## 0. 档案头
- 建档时间
- 当前版本
- 建档动机（一句话）
- 完整度自评：高 / 中 / 低

## 1. 基础信息
- 年龄阶段、教育、职业、居住地
- 重要身份标签
- 成长背景（一段，50字以内）
- 一句话画像（观察者视角）

## 2. 核心特征
### 长期稳定的思维方式
### 长期稳定的价值倾向
### 长期稳定的关系模式
### 已确认的边界与禁区
  （含深度内容，以「自述：」标注来源）

## 3. 关键事件索引
- 时间线（事件 + 日期/阶段 + 一句影响）
- 重要人物（姓名 / 关系 / 一句备注）

## 4. 原始语料索引
- 5–10 条最有档案价值的原话 + 所在章节
- 不全文粘贴 intake

## 5. 档案关键词
- 5–8 个，用于快速定位这个人
```

---

##### 文件二：时期快照（动态层）

> 更新频率：每次建档 / 每次追问后更新
> 目的：「这个人现在在哪里」

```
# 时期快照（动态层）

## 快照信息
- 快照日期
- 距上次更新：X 天
- 本次更新触发原因

## 1. 当前处境
- 生活状态（一句）
- 当前关注重点
- 当前关键词（3–5 个）
- 当前主要压力源

## 2. 当前主要张力
- 内在张力（自述 + 观察）
- 外在张力（自述 + 观察）

## 3. 近期重要事件
- 事件 + 日期 + 一句影响（新增到核心档案时间线）

## 4. 正在探索的问题
- 列条目，标注「本人知晓」/ 「本人尚未命名」

## 5. 本次建档新增 / 修订
- 对核心档案的修订建议（供人工确认后写入）
```

---

#### 改动 3：修复目录冗余

现有目录中 §5「娱乐与内容偏好」和「金钱花费方向」各自在独立节重复出现。在新结构中：

- **娱乐偏好、审美、消费观**——归入核心档案 §2「核心特征」下的「长期稳定的价值倾向」，以条目形式记录
- **当前兴趣、当前消费重点**——归入时期快照 §1「当前处境」

不再设独立的「能力与生活」大节，技能和生活习惯作为补充档案（插件）单独管理，主档案不重复。

---

#### 改动后的完整 System 提示（产物 B）

```
你是人物档案的整理者。任务：从 L1 分析中间层 抽取并转写为 产物 B：可长期更新的人物档案。

产物 B 由两个文件构成：
- 核心档案（稳定层）：骨架信息，更新频率低
- 时期快照（动态层）：当前状态，每次建档后更新

### 两个文件的目录（必须遵守）
{CATALOG_B}

### 写作规则

1. 按目录各级标题输出；每节用 bullet 或短段落，避免长叙事。

2. 优先 L1 中自述事实与已确认内容。

3. 【来源标注原则】
   - 可观察事实：直接陈述
   - 填写者自述：加「自述：」前缀
   - 分析者归纳：加「观察：」前缀
   禁止：把填写者的自我解读直接写成客观陈述。

4. 核心档案 §0 须含：快照日期、建档动机、完整度（高 / 中 / 低）。

5. 核心档案「原始语料索引」：只列 5–10 条最有档案价值的原话 + 对应章节，不全文粘贴 intake。

6. 时期快照须含「本次建档新增 / 修订」一节，列出对核心档案的修订建议（供人工确认）。

7. 不用 MBTI 标签；类型学仅在附录可选且标注「参考」。

8. 缺料写「尚未记录」，不阻断。
```

## 三、后续可以考虑（现在不做）

- **隐私双版本**：核心报告 + 可对外分享的删减版。现阶段分级节（深度观察）已提供最小化的隐私保护，完整双版本等系统稳定后再设计。
- **档案插件体系**：技能档案、健康档案、学习档案等补充档案的标准格式。目前主档案已预留「关联插件」列表位置，正式格式等追问设计完成后统一规划。
- **篇幅上限规则**：给模型加字数约束（如核心档案 800 字以内、时期快照 400 字以内）。需要跑实验确认合理数字后再写入提示词。
- **第三人称分享版报告**：当用户需要把分析报告给他人看时，需要一份第三人称 + 裁掉深度观察节的版本。提示词改动简单，但触发逻辑需要产品层支持。

## 四、改动优先级

| 改动 | 优先级 | 成本 | 收益 |
|------|--------|------|------|
| 产物 A：强制第二人称 | 高 | 极低（一行规则） | 直接提升用户阅读体验 |
| 产物 A：深度内容分级 | 高 | 低（新增一条规则） | 解决敏感内容暴露问题 |
| 产物 B：来源标注原则 | 高 | 低（新增一条规则） | 修复档案的根本性错误 |
| 产物 B：稳定层/动态层拆分 | 中 | 中（需重写目录） | 降低长期维护成本，改善阅读重心 |
| 产物 B：目录冗余修复 | 低 | 极低 | 减少生成冗余 |
