### 1. 人称与口吻

**在问什么**  
读报告的人，感受到的是「一份客观档案」，还是「某位分析者在向读者介绍这个人」？


| 选项              | 读感示例                     | 优点                                                                              | 代价                     |
| --------------- | ------------------------ | ------------------------------------------------------------------------------- | ---------------------- |
| **A. 纯第三人称档案体** | 「他目前就读于…」「在关系里，他更看重…」    | 与 [person-archive-engine.md](../person-archive-engine.md) 目标一致；日后 Renderer 默认好做 | 你的「分析体系」色彩较弱           |
| **B. 分析者视角**    | 「从材料看，他对思考的依赖…」「值得注意的是…」 | 显式承载你的认识论；适合作为方法论示范                                                             | 与「档案像一个人」略远；需约定何时出现分析者 |
| **C. 混合**       | 正文 A，§13 或段末短评用 B        | 兼顾可读与体系                                                                         | 要划清边界，否则口吻漂移           |


**为何重要**  
你提到档案本质也在讲「自己的分析体系」。若选 B 或 C，§13「综合观察」与正文的语气分工必须事先说好，否则同一份报告会像两个人写的。

**当前**：【B，分析本身就是和档案相辅相成的，按需要即可，刻意用档案体约束不会有好处】

### 2. 「询问空间」粒度

**在问什么**  
AI 初稿里，哪些地方**故意留给你改、问你、或标出推断**，而不是装成定论？

这是为 **你二次定稿** 留的接口，不是给最终读者看的功能（定稿时你会删掉或改写大部分）。


| 机制                  | 放什么                    | 适合何时用                           |
| ------------------- | ---------------------- | ------------------------------- |
| **章末 `> **待确认**：`** | 整章读完后仍不确定的点（语料冲突、多种读法） | 料足但interpretation 分叉的章（如 §4、§7） |
| **正文内联 `〔推断〕`**     | 单句：有依据但非用户原话           | 从多处答案拼出的结论                      |
| **正文内联 `〔待你确认〕`**   | 单句：依据不足，需你拍板           | 你卷里自己标了 Q1/Q2 类模糊处              |
| **文末「开放问题」**        | 5–10 个直接问你的话           | 留空题、跳框架回答、敏感处                   |


还要定 **密度**：

- **密**：每章末都有待确认 → 你改起来省力，但初稿难读  
- **疏**：仅文末汇总 → 正文流畅，你改时要自己找疑点  
- **折中**（常见）：正文少量 `〔推断〕`，仅 **§5、§7、§8、§13** 章末加待确认，文末 5 条开放问题

**当前**：【按需要，一切以内容为导向】

### 3. §13 综合观察：与正文的边界

**在问什么**  
catalog §13 是「跨章节整体印象」，不由问卷直出。写分析报告时：

- **正文 §2–§12**：按章写「这个人这一块是什么样」  
- **§13**：写「拼在一起像什么样的人」——允许归纳、对照、点出矛盾是否自洽

需要避免两种失败：

1. **§13 重复正文**：把 §7 关系段再抄一遍
2. **§13 跳太远**：正文没写的「人格类型」或医疗式诊断

还要定 **谁写 §13**：


| 方式                                | 说明                                           |
| --------------------------------- | -------------------------------------------- |
| **与 §1–§12 同一次起草**                | 人读连贯；易与前面重复                                  |
| **假定 §1–§12 已定，再 synthesize §13** | 更贴近日后引擎（`ai_synthesize`）；与引擎文档一致             |
| **§1.1 核心印象与 §13 一起 synthesize**  | 1.1 不宜简单拼接字段；你现已补 1.1 原话，但仍可 synthesize 成档案句 |

| 策略         | 写法                | 何时用         |
| ---------- | ----------------- | ----------- |
| **尚未记录**   | 该节一句带过或目录性省略      | 用户未答、且不重要   |
| **写入开放问题** | 正文不写，文末问你         | 重要但缺料，且你愿意补 |
