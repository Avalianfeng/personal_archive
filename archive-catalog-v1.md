# 理想人物档案目录 V1（初稿）

> **性质**：产品与设计初稿，非最终实现规格。  
> **范围**：只回答「读完这份档案，应该了解这个人哪些方面」——不写问题、不写数据库、不写 Processor。  
> **关联**：[docs/person-archive-engine.md](docs/person-archive-engine.md) · [gpt.md](gpt.md)

---

## 使用说明

本文档分四块，阅读时注意标记：

| 标记 | 含义 |
| --- | --- |
| **【读者可见】** | 最终报告（详细模式）中应呈现的内容结构 |
| **【仅 Meta，不展示】** | 进入档案引擎的语境数据，影响解读与权重，一般不出现在正文 |
| **【设计说明】** | 给产品/实现的内部备注，V1 可不写入用户报告 |
| **【附录可选】** | 有材料才展开，无则省略整节 |

层级约定：**一级 = 大章 · 二级 = 节 · 三级 = 条**（三级是「应了解的具体方面」，不是问卷题目）。

---

## 核心原则（全文约束）

1. **档案是关于人的，不是关于问题的。** 目录每一节都在回答「了解这个人」，而不是「第几题放了什么」。
2. **本产品首先是建档，不是测评。** 不出现「你是某某型人格」作为档案主体；类型学标签仅可进附录作参考视角。
3. **存储可分客观/人格，阅读应像一个人。** 正文不按「客观卷 / 人格卷」分册，而按人生主题组织叙述。
4. **所有采集插件（表单、人格深潜、专业问卷）的内容，都必须能归入本目录的某一三级条。** 归不进去的，要么扩目录，要么删插件。

---

## 产品范围（V1 定位）

本目录描述的是 **「人」的详尽档案本体**——读完应像认识一个完整的人，而非读完一份技能清单或测评报告。

| 层次 | 范围 | 说明 |
| --- | --- | --- |
| **核心（本目录 §0–§13）** | 人的详尽档案 | 建档、人格叙述、关系、价值观、轨迹、未来等；**个人档案与人格报告均以此为准** |
| **专项插件（外挂）** | 技能、收入、行业问卷等 | 独立采集，结果挂载到相关三级条（如 §9 能力）；**不并入主卷题量**，不替代本目录结构 |
| **人格深潜等人格探索** | 沉浸式探测插件 | 向量与叙述摘要进附录或汇入正文相关条；**不以插件分册替代正文叙事** |
| **简略模式** | 同一目录的子集 | 从详尽档案中 **选取** 呈现（加权、裁剪、摘要），**不另建一套更短的目录** |

修订原则：一级章（§1–§13）与 Meta 层相对稳定；**三级条**可随验证增删改；专项能力不进正文新章，只扩展 §9 等既有条或附录。

# 第二部分 【设计说明】Frame / Moment 与正文目录的关系

> 本节不展示给普通读者。说明 Meta 如何作用于正文，避免「问了一百题不知道放哪」。

## 2.1 默认目录 vs 加权目录

- **默认**：下列第三部分的全部一级章，构成「全量理想档案」。
- **加权**：根据 `0.2 Frame` 与 `0.3 Moment`，对三级条标记 `核心 / 标准 / 弱化 / 跳过`，报告生成时：
  - **核心**：充分展开，可多段叙述
  - **标准**：有材料则写，无则一句带过或「尚未记录」
  - **弱化**：娱乐向动机时，如「人生轨迹」可缩为短段
  - **跳过**：无材料且 Frame 表明用户无意填，不出现在正文

## 2.2 Frame 对权重的示例（初稿规则，待验证）

| Frame 信号 | 倾向强化的章节 | 倾向弱化的章节 | 报告语气 |
| --- | --- | --- | --- |
| 认真建档 + 尽量真实 | 2–9 全文 | 附录标签 | 观察式、克制 |
| 娱乐玩玩 + 角色扮演 | 10 兴趣审美、13 综合（轻松） | 8 情感深层、11 人生轨迹 | 轻松、注明「含表演成分」 |
| 主要来测人格 | 13 综合、附录人格参考 | 3 基础画像细节表 | 引导至人格深潜插件，避免 MBTI 口吻入主文 |
| 帮他人了解我 + 职场受众 | 3 基础、9 能力、5 价值观 | 8 情感脆弱点 | 偏正式、可对外 |
| Moment：变动/危机 | 2 当下处境、12 未来 | 10 兴趣 | 强调「阶段性快照」免责声明 |

## 2.3 与 Processor 的衔接（概念）

- Meta **不经过**「转写」类 Processor，仅 `normalize` 入库。
- 正文每条三级内容，来源可以是：规则字段、AI 转写、跨条 synthesize、插件模块汇入。
- **同一条三级**可合并多源（例：`9.2.1 职业方向` ← 表单 + 人格深潜 + 技能问卷）。

## 2.4 阅读顺序 vs 存储顺序

- 存储：可按 `objective` / `personality` / `modules` 分桶。
- 阅读：按下文第三部分的**叙事顺序**渲染，客观与人格在同章内交织。
