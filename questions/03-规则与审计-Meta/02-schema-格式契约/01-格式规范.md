## Agent 导入契约（jsonl）

每行一个 JSON 对象。完整字段表见 [00-整理规范-v1.0.md §jsonl 字段契约](../00-整理规范-v1.0.md#jsonl-字段契约)。

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `question` | 是 | 题干 |
| `category` | 是 | slug，与 `02-问题地图-Views/*.md` 对应 |
| `subcategory` | 是 | sync 写入 `##` 标题 |
| `type` | 是 | 见下表 |
| `source` | 建议 | 来源名 |
| `interaction` | 建议 | 回答交互方式 |
| `options` | 条件 | `agreement` / `single` / `multi` 必填 |
| `scale_note` | 否 | `type: scale` 时可选 |
| `tags` / `prerequisites` | 否 | 须登记于 registries yaml |

Agent **不提供** `uid` / `id` — ingest 自动生成。

### type · options · scale_note

| type | 含义 | options | scale_note |
| --- | --- | --- | --- |
| `open` | 开放叙述 | — | — |
| `fill` | 填空（题干含 `_____`） | — | — |
| `agreement` | 认同度 / 「像我」 | **必填** `[{key,text},…]` | — |
| `single` / `multi` | 单选 / 多选 | **必填** | — |
| `scale` | 数值量表 | 可选 | 常用 |
| `sort` | 排序 | 视题型 | — |

### options 格式（与 schema 一致）

```json
"options": [
  {"key": "1", "text": "完全不像我"},
  {"key": "2", "text": "不太像我"}
]
```

- `key`：展示用序号/字母（`"1"`…`"6"` 或 `"A"`…）
- `text`：选项文案
- **禁止** `["完全不像我", "不太像我"]` 字符串数组 — ingest 拒绝，sync 无法渲染

`ingest.py` 入库前校验 options 形状；详见 `db.validate_options`。

## uid 与 id

| 字段 | 角色 |
| --- | --- |

| `id` | 人类编号 `Q-VAL-001`；**删题不重排** |

## 字段参考（DB / sync 视图）

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `category` | 是 | slug |
| `subcategory` | 是 | |
| `type` | 是 | 见上表 |
| `status` | 是 | active · candidate · deprecated |
| `prerequisites` | 否 | 语义严格前提；见 [prerequisites.yaml](../01-registries-规则真源/prerequisites.yaml) |
| `interaction` | 建议 | story · rating · reflection 等 |
| `source` | 建议 | |
| `tags` | 否 | 见 [tags.yaml](../01-registries-规则真源/tags.yaml) |
| `related` | 否 | 题号或 uid |
| `validation` | 否 | 诚实度校验 |

**禁止**：difficulty · mapsTo · dimension · trait · guide

### prerequisites vs tags

| | prerequisites | tags |
| --- | --- | --- |
| 含义 | **无此事实则问题不存在** | 题面涉及什么主题 |
| 严格度 | 极高；绝大多数题省略 | 可逐步补充 |
| 例子 | `has_current_partner` | `relationship` |

宗教 / 政治：**不新开一级分类**；按功能入现有 category，用 `tags` 检索。

## 健康检查

```bash
python questions/scripts/manage.py sync    # DB → categories + export
python questions/qcli.py doctor          # 区分 STALE（未 sync）与 DATA ERROR
```

`doctor` 报 `STALE` → 运行 `manage sync`；报 `ERROR: Q-xxx: options` → 修正 DB 或 jsonl 重导。
