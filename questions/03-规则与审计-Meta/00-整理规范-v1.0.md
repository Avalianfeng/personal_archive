## 定位

人物档案引擎的**知识层**（非问卷系统）：

- Level 1：`01-文献与来源-Library/` 文献馆 — 永不 bulk 导入
- Level 2：`04-存储层-Store/` 题目仓库
- 规则：`registries/*.yaml`
- 导航：`02-问题地图-Views/`（sync 视图）
- 操纵：`qcli.py` · `ingest.py`

## Agent 整理产出

输出 → `05-导入队列-Imports/01-pending-待入库/{source}-{date}.jsonl`（每行 JSON）或 `qcli ingest --stdin`。

Agent **不负责**：`uid`、`id`、`timestamps`（由 `ingest.py` 自动生成）。

### jsonl 字段契约

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `question` | 是 | 题干（中文优先） |
| `category` | 是 | slug：`real` · `emo` · `dec` · `sta` · `self` · `val` · `oth` |
| `subcategory` | 是 | 子分类名；sync 时自动写入 `02-问题地图-Views/*.md` 的 `##` |
| `type` | 是 | 见下表 |
| `source` | 建议 | 来源名，如 `PVQ` · `StoryCorps` |
| `interaction` | 建议 | `story` · `rating` · `reflection` 等 |
| `status` | 否 | 默认 `active`；质量待观察用 `candidate` |
| `options` | 条件 | `agreement` / `single` / `multi`：**内联**对象数组 **或** 用 `options_ref` |
| `options_ref` | 条件 | 引用 `01-registries-规则真源/options_templates.yaml` 中的模板 id |
| `scale_note` | 否 | 量表端点文字说明（`type: scale` 时可用，与 options 二选一） |
| `tags` | 否 | 须登记于 `01-registries-规则真源/tags.yaml` |
| `prerequisites` | 否 | 须登记于 `01-registries-规则真源/prerequisites.yaml` |
| `related` | 否 | 相关题号或 uid |
| `validation` | 否 | 诚实度校验题 → `true` |

**禁止字段**：`difficulty` · `mapsTo` · `dimension` · `trait` · `guide`

#### type 与 options

| type | 何时用 | options |
| --- | --- | --- |
| `open` | 开放叙述 | 不需要 |
| `fill` | 题干含 `_____` 填空指称 | 不需要 |
| `agreement` | 「像我/认同度」Portrait 陈述 + 分级 | **必填** 对象数组 |
| `single` / `multi` | 完整选项列表 | **必填** 对象数组 |
| `scale` | 数值量表（如 1–10） | 可用 `options` 或 `scale_note` |

#### options 格式（硬性）

必须为 **`[{"key":"…","text":"…"},…]`** 对象数组。**禁止**字符串数组（ingest 会拒绝）。

**量表批次省 token**：可用 `options_ref` 引用 registry 模板（`qcli registry list options_templates`）。内联 `options` 与 `options_ref` 同时存在时，**内联优先**。

**可选 batch 首行**（`_meta`，不入库）：

```json
{"_meta": true, "source": "PVQ", "default_options_ref": "portrait_6l_zh"}
```

后续题若 type 需要 options 且未内联，则取行内 `options_ref` → batch `default_options_ref` → 报错。

开放题示例：

```json
{"question":"小时候最常相处的人是谁？","category":"real","subcategory":"家庭与成长","type":"open","interaction":"story","source":"StoryCorps"}
```

认同度量表示例（PVQ 6 级，推荐 options_ref）：

```json
{"_meta": true, "source": "PVQ", "default_options_ref": "portrait_6l_zh"}
{"question":"想出新颖点子、用原创方式做事，对你有多重要？","category":"val","subcategory":"个人价值观","type":"agreement","interaction":"rating","source":"PVQ"}
```

内联 options 仍可用（小批次或非标准量表）：

```json
{"question":"想出新颖点子、用原创方式做事，对你有多重要？","category":"val","subcategory":"个人价值观","type":"agreement","interaction":"rating","source":"PVQ","options":[{"key":"1","text":"完全不像我"},{"key":"2","text":"不太像我"},{"key":"3","text":"有一点像我"},{"key":"4","text":"有些像我"},{"key":"5","text":"像我"},{"key":"6","text":"非常像我"}]}
```

单选示例：

```json
{"question":"朋友连续爽约三次，你更可能？","category":"emo","subcategory":"友谊","type":"single","interaction":"scenario","source":"示例","options":[{"key":"A","text":"直接问清楚"},{"key":"B","text":"减少联系"},{"key":"C","text":"当作偶然"}]}
```

## 量表类来源整理 checklist

适用于 PVQ、RSES、WHOQOL、Big Five 等整份量表：

1. 读 `01-文献与来源-Library/[待整理][Px] *.md` 卡片中的整理注意
2. **去格式** — 剥离网页说明、评分指引；第三人称 portrait → 第二人称（见 [整理Agent启动.md](./04-prompts-Agent提示/01-整理Agent启动.md) §Portrait 改写）
3. **归类** — 按「主要问什么」选 `category`；整批同一主题可用同一 `subcategory`（如 PVQ → `个人价值观`）
4. **type** — Portrait「像我」→ `agreement` + `interaction: rating`；Likert 陈述 → 同上；纯数值端点 → `scale`
5. **options** — 整批统一量表优先 `default_options_ref: portrait_6l_zh`（PVQ）；或内联 6 级 key `"1"`…`"6"`
6. **system_unaskable** — 逐题判断（量表通常无）
7. 输出 jsonl → 跑批次验收
8. 回写来源卡片：状态、题数、id 范围、processed 文件名

## 子分类命名

| 策略 | 适用 | 示例 |
| --- | --- | --- |
| **整批统一** | 同质量表、同一协议 | PVQ 40 题 → `个人价值观` |
| **按人生阶段** | 人生故事访谈 | `青少年时期` · `为人父母` |
| **按功能主题** | 访谈清单分散主题 | `家庭与成长` · `人生目标` |
| **复用已有** | 与 categories 现有 `##` 语义一致 | 优先查 `02-问题地图-Views/*.md` 再决定是否新建 |

新增 `##` 由 jsonl 的 `subcategory` 字段触发 sync 自动写入，**禁止**手改 `02-问题地图-Views/*.md`。

命名要求：中文、2–8 字、描述「这批题共同问什么」，不用来源名（如不用 `PVQ` 作子分类）。

## 批次验收（收集期）

**推荐（一键）**：

```bash
python questions/scripts/ingest.py --dry-run
python questions/scripts/manage.py accept --json
```

`accept` = 校验 pending jsonl → ingest → sync_categories → export_json → **export_agent_views** → doctor。

Agent 读后确认：`03-generated-审计产物/01-agent-Agent视图/batch_delta_compact.md` · `batch_delta.json` · `health.json`。

**清理期**（收集完成后）：`manage sync --with-duplicate-scan` → 查重/审查 Agent，见 [04-prompts-Agent提示/04-清理期Agent启动.md](./04-prompts-Agent提示/04-清理期Agent启动.md)。

**分步**：

1. `python questions/scripts/ingest.py --dry-run`
2. `python questions/scripts/ingest.py`（仅写 DB）
3. `python questions/scripts/manage.py sync`
4. `python questions/qcli.py doctor`（默认 JSON；人类加 `--human`）
5. `05-导入队列-Imports/01-pending-待入库/` 为空（仅 README）

Agent 启动清单：[04-prompts-Agent提示/01-整理Agent启动.md](./04-prompts-Agent提示/01-整理Agent启动.md)

## 失败恢复

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| ingest 报错 `options[N] is a string` | jsonl 用了字符串数组 | 改为 `{key,text}` 对象后重跑 ingest |
| ingest 成功但 sync 崩溃 | 历史版本未校验 options | 修正 jsonl → `qcli edit` 批量修 DB，或删题后重 ingest |
| doctor 报 `STALE: 02-问题地图-Views/… hash drift` | DB 已更新、categories 未 sync | `python questions/scripts/manage.py sync` |
| doctor 报 `ERROR: Q-xxx: options…` | DB 内 options 格式非法 | 用 `qcli edit` 修正或修复 jsonl 重导 |
| `05-导入队列-Imports/01-pending-待入库/` 非空 | ingest 未完成或失败 | 读 `05-导入队列-Imports/03-failed-失败/*.error.log` 修复后重跑 |

ingest 在单文件内**事务提交**；若 `post_write_hooks`（sync/export）失败，DB 已写入但 categories 滞后 — **只重跑 `manage sync`**，勿重复 ingest 同一文件（已归档到 `05-导入队列-Imports/02-processed-已入库/`）。
