## 双格式架构

```text
categories/*.md     人类 / Agent 编辑（Git diff 友好）
        ↓ parse_questions.py
generated/questions.json    程序只读（禁止手改，不提交 Git）
```

---

## 两层生命周期（不要混）

| 维度 | 管什么 | 载体 |
| --- | --- | --- |
| **文件生命周期** | 来源未整理、已整理、查重报告、淘汰来源 | `raw/` `categories/` `duplicates/` `rejected/` `canonical/` |
| **问题生命周期** | 题是否活跃、待观察、已弃用 | frontmatter `status:` → JSON |

- `rejected/`：管理**永远进不了 JSON 的来源文件**（残缺 raw）。此处无 Question，`status` 不适用。
- `status` **不含** `rejected`（文件级）、**不含** `canonical`（标准题用 `canonical/` 目录）。
- `categories/`：人类浏览/编辑主轴（按「这道题问什么」）。
- `tags`：跨分类检索轴，不替代目录。

## 每题一块：YAML frontmatter + 正文

```markdown
## 他人评价

---
id: Q-SELF-001
category: self
subcategory: 他人评价
type: open
interaction: reflection
source: StoryCorps
tags:
  - self_image
  - social_perception
depth:
validation: false
related:
  - Q-SELF-088
status: active
---

别人最容易误解你的地方是什么？
```

`## 子分类` 可保留作人类浏览，但 **frontmatter 必须显式写 `subcategory:`**。

## 字段说明

### 现在就做（整理阶段）

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `id` | 是 | `Q-{REAL\|EMO\|DEC\|STA\|SELF\|VAL\|OTH}-{NNN}` |
| `category` | 是 | slug，须与所在文件名一致 |
| `subcategory` | 是 | 如 `亲密关系`、`他人评价` |
| `type` | 是 | 见下表 |
| `interaction` | 建议 | 见下表 |
| `source` | 建议 | 如 `StoryCorps` |
| `validation` | 否 | 默认 `false` |
| `related` | 否 | 相关题号（同主题/变体/重复候选） |
| `status` | 是 | 默认 `active` |

### 预留但允许为空

| 字段 | 说明 |
| --- | --- |
| `tags` | 检索标签，**不是**推断/mapsTo/traits |
| `depth` | `shallow` · `medium` · `deep` 或空；整理员不填，未来引擎推断 |
| `superseded_by` | `status: deprecated` 时指向更好的题 |
| `deprecated_reason` | 弃用说明 |

### 禁止

`difficulty` · `mapsTo` · `dimension` · `trait`

## 枚举

### category slug

| 文件 | slug |
| --- | --- |
| 现实问题.md | `real` |
| 情感问题.md | `emo` |
| 决策问题.md | `dec` |
| 状态问题.md | `sta` |
| 自我认知.md | `self` |
| 价值问题.md | `val` |
| 其他.md | `oth` |

### type

| 值 | 旧标注 |
| --- | --- |
| `open` | 开放 |
| `single` | 单选 |
| `multi` | 多选 |
| `scale` | 量表(N) |
| `sort` | 排序 |
| `fill` | 填空 |
| `agreement` | 认同度 |

### interaction

`story` · `self_report` · `scenario` · `comparison` · `rating` · `reflection` · `future`

### status

| 值 | 含义 |
| --- | --- |
| `active` | 正常可用 |
| `candidate` | 刚整理、质量待观察 |
| `deprecated` | 不推荐继续使用，保留历史 |

## 选项与量表

单选/多选选项写在 frontmatter 下方：

```markdown
---
id: Q-EMO-001
category: emo
subcategory: 亲密关系
type: single
interaction: scenario
source: 示例
status: active
---

朋友连续爽约三次，你更可能：
- A. 直接问清楚
- B. 减少联系
- C. 当作偶然
- D. 看具体情况
```

量表端点写在正文括号内，如 `（1=很难开口 … 5=很容易表达）`。
