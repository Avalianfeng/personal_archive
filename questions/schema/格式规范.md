```text

        ↓ build_questions.py
generated/questions.json | stats.json | duplicate_hints.json | registries.json
```

## uid 与 id

| 字段 | 角色 |
| --- | --- |
| `uid` | **主键**（8 位 hex）；`related` / 未来映射引用 uid |
| `id` | 人类可读编号 `Q-SELF-001`；**删题不重排**，允许空档 |
| `order` | 可选展示顺序，与 id 解耦 |

Builder 缺失 `uid` 时自动生成并写回 md（`--no-write-uid` 可关闭）。

## 系统不可提问（system_unaskable）

题干预设**在场对话者**（采访者↔被采访者），如「你最喜欢**我的**哪段回忆？」。整理阶段**不入 categories**，写入 [`rejected/system_unaskable.md`](../rejected/system_unaskable.md)。

## 问题库 vs 问题引擎 vs 推断（边界）

| 层 | 放什么 | 本阶段 |
| --- | --- | --- |
| **questions/** | 问什么、能否成立（prerequisites） | ✅ |
| **registries/** | 标准 id 仓库 | ✅ |
| **question_guides/** | 怎么问（AAI 控场说明等） | ❌ 暂不做 |
| **analysis_guides/** | 按题预置解读 | ❌ 暂不做；运行时 AI + 资料 |
| **引擎 / 档案** | 事实仓库、动态虚拟文件夹 | 未来 |

## frontmatter 示例

```yaml
---
uid: 9d7e2c5a
id: Q-EMO-050
category: emo
subcategory: 亲密关系
type: open
prerequisites:
  - has_current_partner
interaction: reflection
source: StoryCorps
tags:
  - relationship
depth:
validation: false
related: []
status: active
---

你和现任伴侣最大的分歧是什么？
```

**勿用**单独一行 `---` 作 Markdown 分隔线。

## 字段（v0.4）

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `uid` | Builder 补 | 8 位 hex 主键 |
| `id` | 是 | 人类编号；删题不重排 |
| `category` | 是 | 与文件名 slug 一致 |
| `subcategory` | 是 | |
| `type` | 是 | open · single · multi · scale · sort · fill · agreement |
| `status` | 是 | active · candidate · deprecated |
| `prerequisites` | 否 | **语义严格前提**（无此事实则问题不成立）；**宁缺毋滥**；见 [prerequisites.md](../registries/prerequisites.md) |
| `interaction` | 建议 | story · self_report · scenario · comparison · rating · reflection · future |
| `source` | 建议 | |
| `tags` | 否 | **主题检索**；须登记于 [tags.yaml](../registries/tags.yaml) |
| `depth` | 否 | 可空 |
| `related` | 否 | 题号或 uid |
| `order` | 否 | 展示顺序 |

**禁止**：difficulty · mapsTo · dimension · trait · guide（guide 不进 metadata）

### type 与模板题

题干含 **`_____` 或需填空白指称**（缅怀模板等）→ `type: fill`，不是 `open`。

### prerequisites vs tags

| | prerequisites | tags |
| --- | --- | --- |
| 含义 | **无此事实则问题不存在** | 题面涉及什么主题 |
| 严格度 | 极高；绝大多数题省略 | 可逐步补充 |
| 例子 | `has_current_partner`（题干指称「现任伴侣」） | `relationship` |
| 引擎 | 已知 false → 自动跳过 | 检索、组卷 |

**判定**：被问者说「我没有 X」后，题是否变成空问？否 → 不写 prerequisite。用户可自行跳过「有点怪」的题。

宗教 / 政治：**不新开一级分类**；按功能入现有 category，用 `tags: [religion]` 等检索。

