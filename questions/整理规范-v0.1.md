## 目的

确保问题库进入整理阶段后，能够长期扩展、维护和演化，而不会在后期因分类、去重或格式问题被迫大规模返工。

## 一、当前判断

项目已经完成：**收集阶段**。

正在进入：**整理阶段**。

此时最重要的问题已经不是「有没有题」「题够不够多」，而是：

- 如何保证题库长期可维护
- 如何保证后续持续增长
- 如何保证未来能接入问题引擎

---

## 二、项目定位

问题库**不是**：心理测量库、人格诊断库、精神疾病筛查库。

问题库的**目标**：通过提问帮助**认识一个人**。

因此：量表、访谈、口述史、咨询 intake、人生故事——都只是**问题来源**。来源不重要，**问题本身**重要。

---

## 三、关于全面性

**结论**：不追求收集完整，追求持续增长。

| 错误目标 | 正确目标 |
| --- | --- |
| 收集全世界所有问题 | 建立持续增长的问题发现机制 |

问题库**永远不会完成**，只会**持续迭代**。

来源索引见 [question_sources/](./question_sources/README.md)。

---

## 四、关于分类体系

### 当前一级分类（冻结）

| 文件 | 回答什么 |
| --- | --- |
| 现实问题 | 发生过什么、是什么 |
| 情感问题 | 感受什么、关系体验 |
| 决策问题 | 怎么选择、怎么判断 |
| 状态问题 | 现在怎么样 |
| 自我认知 | 怎么看自己 |
| 价值问题 | 认为什么重要 |
| 其他 | 暂无法归入以上类 |

**冻结规则**：至少在题库达到 **500–1000 题**前，不要新增一级分类。

**原因**：当前分类是**问题功能分类**，不是心理学理论分类。功能分类更稳定；人格理论会变化，功能分类不会。

| 允许 | 不允许 |
| --- | --- |
| 新增 `## 子分类` | 新增 `依恋问题.md`、`创伤问题.md`、`人格问题.md` 等理论类文件 |

分类定义见 [prompts/分类原则.md](./prompts/分类原则.md)。

## 五、关于心理疾病与量表

**建议保留，但降低优先级**。

项目主轴是**认识一个人**，不是**诊断一个人**。

PHQ-9、GAD-7、ASRS、MDQ 等内容可以收录，但定位应为**状态问题**，而不是核心问题来源。

**更高价值来源**：心理咨询 intake、人生故事访谈、口述史、关系访谈。

## 六、整理阶段的去重原则

**不做文本去重**。

| 错误 | 正确 |
| --- | --- |
| 句子不同就保留，句子相似就删除 | 按**信息意图**去重 |

### 三种状态

| 状态 | 处理 |
| --- | --- |
| **保留** | 信息意图不同，正常入库 |
| **重复候选** | 语义相近但可能触发不同记忆——**两题都保留**，标注 `[重复候选: Q-xxx]` |
| **完全重复** | 同义同形——整理阶段可跳过输出，标注 `[完全重复: Q-xxx]` |

**示例**：

- Q-SELF-012：别人最容易误解你的地方是什么？
- Q-SELF-088：别人经常对你有哪些错误印象？

→ 保留两题，并标记 `[重复候选: Q-SELF-012]`。相似表达可能触发不同记忆，对人物建档仍有价值。

## 七、整理流程

**不要边整理边查重**。

```text
raw/
  ↓ 整理 Agent（去格式 → 归类 → 编号）
categories/
  ↓ 查重 Agent（批量）
duplicates/
```

| Agent | 职责 |
| --- | --- |
| **整理 Agent** | 去格式、归类、编号、规范化 |
| **查重 Agent** | 发现重复、输出重复报告（不删题） |

## 八、关于题目元信息

在 `###` 标题下可选标注：

| 字段 | 枚举 / 说明 |
| --- | --- |
| **题型** | 开放 · 单选 · 多选 · 量表(N) · 排序 · 填空 · 认同度 |
| **回答形式** | `story` · `self_report` · `scenario` · `comparison` · `rating` |
| **来源** | 如 `StoryCorps`、`McAdams Life Story` |
| **[重复候选]** | 题号 cross-ref |
| **[校验]** | 诚实度/一致性校验 |

**示例**：

```markdown
### Q-VAL-023 · 单选
回答形式: comparison
来源: StoryCorps
[重复候选: Q-SELF-012]

自由与安全只能保留一个，你会选哪个？
```

**意义**：未来问题引擎可控制节奏，避免连续二十个开放题导致疲劳。

## 十五、02-维度地图 · 严格隔离（最重要）

**整理 Agent 不得接触 [02-维度地图-dimensions.md](../02-维度地图-dimensions.md)。**

```text
raw → categories     ✅ 整理阶段
02-维度地图          ❌ 整理阶段禁止
```

**原因**：许多项目死在这里——收集问题 → 立即映射维度 → 发现维度不合理 → 重新整理全部题库。

本项目优势：**问题层 · 推断层 · 档案层** 已拆开。

至少在整理完成、达到几百题之前：

```text
问题 ≠ 维度
维度 ≠ 档案
```

**整理 Agent 禁止读取或引用**：

- `02-维度地图-dimensions.md`
- `design/catalog/`、`samples/persona-v1/`（档案层）
- `mapsTo`、`evidence_tags`、人格/诊断推断

此边界保证：无论维度地图如何重构，**题库不受影响**。

## 十八、prerequisites 与 tags（v0.4）

| 字段 | 含义 | 仓库 |
| --- | --- | --- |
| `prerequisites` | **无此事实则问题不存在**（语义/指称严格前提） | `registries/prerequisites.yaml` |
| `tags` | 题面**涉及什么主题**（跨类检索） | `registries/tags.yaml` |

**prerequisites 克制原则**：只有题干必要指称/状态缺失会导致**空问**时才写。可答「没有/从未/不适用」、或用户可自行跳过的题 — **不写**。详见 [prerequisites.md](./registries/prerequisites.md)。

**不要用 tag 表达事实**（如 `has_partner` 应写进 prerequisites — 且须过上述严格判定）。

宗教 / 政治：不新开一级 category；按功能分类 + `tags: [religion]` 等。

**本阶段不做**：

- 批量给现有 209 题补 prerequisites/tags
- `question_guides/`（怎么问）
- `analysis_guides/`（按题预置解读 — 运行时 AI + 资料）

未来引擎：事实仓库 → 已知 false 时跳过 → 否则始终可呈现（用户可跳过）。

### registry 按需增长（v0.4）

**写不完，也不该写完。** 「完全」= 当前题库**引用完整**（无 unknown id），非枚举一切人生指称。

整理时**允许**同批追加 `registries/*.yaml`，**必须**经 [registry审核提示词.md](./prompts/registry审核提示词.md) 子 Agent 审核。详见 [registries/README.md](./registries/README.md)。

## 二十一、整理批次验收（P1 归因 · 冻结）

StoryCorps P1 暴露的问题，本质可归因于 **「灌库捷径 > 提示词流程」**。后续批次**必须**走本清单；categories 现有 209 题为练手产物，**不清空**，但新批次不得重复 P1 错误。

| # | 验收项 | 命令 / 动作 |
| --- | --- | --- |
| 1 | **逐题整理** — Agent 按 [问题整理提示词.md](./prompts/问题整理提示词.md) 输出 frontmatter；**禁止** bulk 脚本写 categories | 人工确认无新 `scripts/*_pN.py` 灌库 |
| 2 | **system_unaskable** — 逐题筛 dyadic | `python questions/scripts/build_questions.py --audit-dyadic` |
| 3 | **编译 + registry** | `python questions/scripts/build_questions.py --strict-registry` |
| 4 | **查重** — 语义近重复（非整理阶段职责） | [问题查重提示词.md](./prompts/问题查重提示词.md) → `duplicates/` |
| 5 | **新 id 编号** | 各类从 max+1 起；不复用空档 |
| 6 | **type** | 含 `_____` 模板 → `fill`，非 `open` |
| 7 | **registry 新 id** | [registry审核提示词.md](./prompts/registry审核提示词.md) 子 Agent |
| 8 | **raw 队列** | 移 `processed/`；来源卡片改 `[已整理]` |
| 9 | **git** | 工程层 commit；raw 正文不进 Git（见 `.gitignore`） |

**metadata  conscious 缺失**（tags / prerequisites 可后续补）不阻断批次，但须在验收报告中注明。
