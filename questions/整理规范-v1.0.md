# questions/ 整理规范 · v1.0

> **状态**：DB 时代 · 2026-06-16  
> **关联**：[README.md](./README.md) · [prompts/问题整理提示词.md](./prompts/问题整理提示词.md) · [schema/格式规范.md](./schema/格式规范.md)

## 定位

人物档案引擎的**知识层**（非问卷系统）：

- Level 1：`source_library/` 文献馆 — 永不 bulk 导入
- Level 2：`question_registry/` 题目仓库
- 规则：`registries/*.yaml`
- 导航：`categories/`（sync 视图）
- 操纵：`qcli.py` · `ingest.py`

## Agent 整理产出

输出 → `imports/pending/{source}-{date}.jsonl`（每行 JSON）或 `qcli ingest --stdin`。

Agent **不负责**：`uid`、`id`、`timestamps`（由 `ingest.py` 自动生成）。

### jsonl 字段契约

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `question` | 是 | 题干（中文优先） |
| `category` | 是 | slug：`real` · `emo` · `dec` · `sta` · `self` · `val` · `oth` |
| `subcategory` | 是 | 子分类名；sync 时自动写入 `categories/*.md` 的 `##` |
| `type` | 是 | 见下表 |
| `source` | 建议 | 来源名，如 `PVQ` · `StoryCorps` |
| `interaction` | 建议 | `story` · `rating` · `reflection` 等 |
| `status` | 否 | 默认 `active`；质量待观察用 `candidate` |
| `options` | 条件 | `agreement` / `single` / `multi` **必填**；见下 |
| `scale_note` | 否 | 量表端点文字说明（`type: scale` 时可用，与 options 二选一） |
| `tags` | 否 | 须登记于 `registries/tags.yaml` |
| `prerequisites` | 否 | 须登记于 `registries/prerequisites.yaml` |
| `related` | 否 | 相关题号或 uid |
| `validation` | 否 | 诚实度校验题 → `true` |

**禁止字段**：`difficulty` · `mapsTo` · `dimension` · `trait` · `guide`

#### type 与 options

| type | 何时用 | options |
| --- | --- | --- |
| `open` | 开放叙述 | 不需要 |
| `fill` | 题干含 `_____` 填空指称 | 不需要 |
| `agreement` | 「像我/认同度」Portrait 陈述 + 分级 | **必填** 对象数组 |
| `single` / `multi` | 完整选项列表 | **必填** 对象数组 |
| `scale` | 数值量表（如 1–10） | 可用 `options` 或 `scale_note` |

#### options 格式（硬性）

必须为 **`[{"key":"…","text":"…"},…]`** 对象数组。**禁止**字符串数组（ingest 会拒绝）。

开放题示例：

```json
{"question":"小时候最常相处的人是谁？","category":"real","subcategory":"家庭与成长","type":"open","interaction":"story","source":"StoryCorps"}
```

认同度量表示例（PVQ 6 级）：

```json
{"question":"想出新颖点子、用原创方式做事，对你有多重要？","category":"val","subcategory":"个人价值观","type":"agreement","interaction":"rating","source":"PVQ","options":[{"key":"1","text":"完全不像我"},{"key":"2","text":"不太像我"},{"key":"3","text":"有一点像我"},{"key":"4","text":"有些像我"},{"key":"5","text":"像我"},{"key":"6","text":"非常像我"}]}
```

单选示例：

```json
{"question":"朋友连续爽约三次，你更可能？","category":"emo","subcategory":"友谊","type":"single","interaction":"scenario","source":"示例","options":[{"key":"A","text":"直接问清楚"},{"key":"B","text":"减少联系"},{"key":"C","text":"当作偶然"}]}
```

## 量表类来源整理 checklist

适用于 PVQ、RSES、WHOQOL、Big Five 等整份量表：

1. 读 `source_library/[待整理][Px] *.md` 卡片中的整理注意
2. **去格式** — 剥离网页说明、评分指引；第三人称 portrait → 第二人称（She → 你）
3. **归类** — 按「主要问什么」选 `category`；整批同一主题可用同一 `subcategory`（如 PVQ → `个人价值观`）
4. **type** — Portrait「像我」→ `agreement` + `interaction: rating`；Likert 陈述 → 同上；纯数值端点 → `scale`
5. **options** — 6 级量表 key 用 `"1"`…`"6"`；单选用 `"A"`… 或 `"1"`…，整批统一
6. **system_unaskable** — 逐题判断（量表通常无）
7. 输出 jsonl → 跑批次验收
8. 回写来源卡片：状态、题数、id 范围、processed 文件名

## 子分类命名

| 策略 | 适用 | 示例 |
| --- | --- | --- |
| **整批统一** | 同质量表、同一协议 | PVQ 40 题 → `个人价值观` |
| **按人生阶段** | 人生故事访谈 | `青少年时期` · `为人父母` |
| **按功能主题** | 访谈清单分散主题 | `家庭与成长` · `人生目标` |
| **复用已有** | 与 categories 现有 `##` 语义一致 | 优先查 `categories/*.md` 再决定是否新建 |

新增 `##` 由 jsonl 的 `subcategory` 字段触发 sync 自动写入，**禁止**手改 `categories/*.md`。

命名要求：中文、2–8 字、描述「这批题共同问什么」，不用来源名（如不用 `PVQ` 作子分类）。

## 批次验收

1. `python questions/scripts/ingest.py`
2. `python questions/scripts/manage.py sync`
3. `python questions/scripts/duplicate_scan.py`
4. `python questions/qcli.py doctor`
5. `imports/pending/` 为空（仅 README）

## 失败恢复

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| ingest 报错 `options[N] is a string` | jsonl 用了字符串数组 | 改为 `{key,text}` 对象后重跑 ingest |
| ingest 成功但 sync 崩溃 | 历史版本未校验 options | 修正 jsonl → `qcli edit` 批量修 DB，或删题后重 ingest |
| doctor 报 `STALE: categories/… hash drift` | DB 已更新、categories 未 sync | `python questions/scripts/manage.py sync` |
| doctor 报 `ERROR: Q-xxx: options…` | DB 内 options 格式非法 | 用 `qcli edit` 修正或修复 jsonl 重导 |
| `imports/pending/` 非空 | ingest 未完成或失败 | 读 `imports/failed/*.error.log` 修复后重跑 |

ingest 在单文件内**事务提交**；若 `post_write_hooks`（sync/export）失败，DB 已写入但 categories 滞后 — **只重跑 `manage sync`**，勿重复 ingest 同一文件（已归档到 `imports/processed/`）。

## status

| 层 | 值 |
| --- | --- |
| 题目（DB） | active · candidate · deprecated |
| 来源淘汰 | `rejected/`（不进 DB） |

## 禁止

- ingest 扫描 `source_library/`
- 手改 `categories/*.md`
- 手改 `registries/*.md`（已删除，仅 yaml）

## 来源优先级（人类排期参考）

| 优先级 | 来源类型 | 归类提示 |
| --- | --- | --- |
| **P1** | 人生故事访谈、StoryCorps、口述史、AAI | `story`；现实 / 情感 / 自我认知 |
| **P2** | PVQ、RSES、WHOQOL 等 | 价值 → `val`；自尊 / 生活质量 → `sta` 或 `self` |
| **P3** | Big Five、MBTI | `rating` / `agreement`；分散入各类 |
