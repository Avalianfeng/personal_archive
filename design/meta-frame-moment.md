# 档案语境层：Frame 与 Moment（草案）

> 配套 [02-档案目录.md](../02-档案目录.md) 第一部分「0. 档案元信息」。  
> **【仅 Meta，不展示】** — 影响解读与章节权重，一般不进入报告正文。

---

## 1. 为什么要单独一层

用户进入档案时，心智可能完全不同：

| 用户心想的是 | 若按同一套报告处理会怎样 |
| --- | --- |
| 我就是来测测人格 | 报告像测评，不像档案 |
| 我想认真留一份给自己的记录 | 需要深度、可回溯 |
| 随便玩玩 | 需要轻松语气 + 免责声明 |
| 填给伴侣/老板看 | 需克制私密、突出可对外信息 |

因此产品在**任何表单之前**，应先收集 Frame 与 Moment，写入 `PersonModel.meta`，供 Processor 与综合章节读取。

**对用户的定位语（向导首屏）**：

> 你在建立自己的档案，不是在参加测试。  
> 下面两个问题帮助我们理解：你为何在此、你此刻的状态——它们会改变档案的写法，但不会作为「人格结论」出现在正文里。

## 2. Frame · 你如何认识这次问答

### 2.1 建立动机（单选 + 可补充）

| 选项 key | 标签 | 内部备注 |
| --- | --- | --- |
| `archive-serious` | 认真建档，留给未来的自己 | 默认权重：全文标准 |
| `explore-self` | 想更了解自己 | 强化 4–8、13 章 |
| `for-others` | 想让别人更了解我 | 强化 3、5、9；弱化 8.3 |
| `entertainment` | 娱乐、好奇、玩玩 | 弱化 11、8；语气轻松 |
| `test-driven` | 主要是想做测试 | 引导人格深潜；附录加重 |
| `other` | 其他（请说明） | 读自由文本 |

### 2.2 与「测试」的关系（单选）

| 选项 key | 标签 |
| --- | --- |
| `not-test` | 我不把这当成测试 |
| `test-plus-more` | 测试是一部分，但我想要更完整的档案 |
| `test-only` | 我主要想要测试结果 |
| `unsure` | 还没想好 |

**与建立动机的交叉**：`test-driven` + `test-only` 时，报告开篇应明确「本档以探索体验为主，完整建档建议继续填写通用档案」。

### 2.3 诚实度预期（单选）

| 选项 key | 标签 | 报告影响 |
| --- | --- | --- |
| `honest` | 尽量真实 | 默认 |
| `polished` | 会美化或包装自己 | 综合章可注「自述含理想化成分」 |
| `roleplay` | 当角色扮演 / 虚构人设 | 显著免责声明；附录标「表演性」 |

### 2.4 时间尺度（单选）

| 选项 key | 标签 |
| --- | --- |
| `moment` | 记录此刻的我 |
| `enduring` | 概括一贯的我 |
| `mixed` | 两者都有 |

影响 2.4 快照说明、13.3 变化趋势的写法。

### 2.5 受众想象（多选或单选）

| 选项 key | 标签 |
| --- | --- |
| `self` | 只给自己看 |
| `close` | 伴侣、家人、好友 |
| `work` | 同事、上级、职场 |
| `public` | 可能公开 |
| `unsure` | 不确定 |

`work` 选中时：8.3 脆弱点、11.2 未竟之事等默认弱化或需用户二次确认才写入正文。

### 2.6 自由补充

开放题，placeholder：「用一句话说说，你怎么看待这次填写。」

## 3. Moment · 你此时的状态

### 3.1 生活阶段快照（单选）

| 选项 key | 标签 |
| --- | --- |
| `stable` | 相对稳定 |
| `transition` | 正在变动（换工作、搬家、关系变化…） |
| `crisis` | 危机或重大转折 |
| `unclear` | 说不清，有点乱 |

### 3.2 情绪底色（单选，可跳过）

| 选项 key | 标签 |
| --- | --- |
| `calm` | 平静 |
| `anxious` | 焦虑 |
| `excited` | 兴奋、充满期待 |
| `low` | 低落 |
| `skip` | 不想谈 |

`low` / `crisis` 时：综合章避免「正能量」口吻；可建议资源提示（非医疗诊断）。

### 3.3 填写精力（单选）

| 选项 key | 标签 |
| --- | --- |
| `focused` | 可以认真填 |
| `quick` | 想快点填完 |
| `minimal` | 打算大部分跳过 |

影响完整度展示与是否推荐「稍后再续填」。

### 3.4 距上次更新（单选，首次建档可隐藏）

| 选项 key | 标签 |
| --- | --- |
| `first` | 第一次建立 |
| `long` | 很久没更新 |
| `recent-event` | 刚经历大事，想更新 |

### 3.5 自由补充

开放题，placeholder：「最近的你，一句话。」

## 4. 前置向导流程（产品草案，非最终实现）

```text
屏 1  定位语 + 「开始建立档案」
屏 2  Frame：建立动机 + 与测试的关系
屏 3  Frame：诚实度 + 时间尺度 + 受众（可合并）
屏 4  Moment：生活阶段 + 情绪底色 + 填写精力
屏 5  两句自由补充（可选）→ 进入整页表单 /form
```

- 每屏可「跳过」→ 对应字段记 `unknown`，报告用默认权重。
- 跳过全部 Meta 时，开篇 1.2 注明「未声明填写语境，按默认建档模式生成」。


## 6. 与正文目录的映射（摘要）

完整权重表见 [02-档案目录.md §2.2](../02-档案目录.md)。

原则：

- Frame 决定 **语气 + 章节权重 + 免责声明**
- Moment 决定 **第 2 章「当下与处境」的详略 + 综合章是否强调「阶段性」**
- 二者均 **不替代** 第 4–12 章的具体内容，只影响如何写、写多深
